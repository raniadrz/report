Technology Choices

Tech Stack — Γιατί αυτές οι τεχνολογίες;

Κάθε επιλογή έχει συγκεκριμένο λόγο. Δεν υπάρχει τίποτα "default" εδώ.

⚛️
React 18
v18.3.1
Η εφαρμογή διαχειρίζεται πολύπλοκο, δυναμικό state: πολλά αρχεία, πολλά sheets, αναλύσεις ανά αρχείο/sheet, loading states, corrections. Το React component model + useState/useCallback κάνουν αυτόν τον συντονισμό χειριστό. Concurrent features δεν χρειάζονται ακόμα, αλλά η βάση είναι εκεί.
Component State useCallback memoization Concurrent Ready
⚡
Vite 5
v5.3.1
HMR σε <50ms κατά το development — κρίσιμο όταν κάνεις fine-tuning UI. Το @e965/xlsx είναι βαρύ package (~1MB). Το Vite χειρίζεται αυτόματα το code splitting και tree-shaking, κρατώντας το bundle μικρό. Το Webpack θα ήταν πολύ πιο αργό στο dev.
Instant HMR ESM Native Tree Shaking
🎨
Tailwind CSS 3
v3.4.4
Το UI έχει πολλές conditional styles (αρχείο ενεργό/ανενεργό, loading/done, header/data row). Με Tailwind αυτά γράφονται inline στο JSX ως template literals χωρίς να χρειαστεί ξεχωριστό CSS file. JIT compiler = μηδενικό unused CSS στο production.
Utility First JIT Compiler Zero Runtime
📊
@e965/xlsx (SheetJS fork)
v0.20.3
Η επίσημη SheetJS αφαίρεσε τη δωρεάν πρόσβαση στα cell styles (bgColor, bold) στις νέες εκδόσεις. Το @e965/xlsx είναι community fork που τα διατηρεί. Αυτά τα styles είναι κρίσιμα — το AI τα χρησιμοποιεί για να εντοπίσει header rows (πολύχρωμες επικεφαλίδες).
Cell Styles bgColor Support Client-side Fork (ιδιαίτερη επιλογή)
🚀
Node.js + Express
Express 4.x
Ο server δεν κάνει τίποτα άλλο εκτός από HTTP proxy στα AI APIs + αποθήκευση corrections. I/O-bound workload = Node.js ιδανικό. Express = minimal overhead. Δεν χρειάζεται database, ORMs, ή complex middleware — γι' αυτό δεν επιλέχθηκε Nest.js ή Fastify.
Minimal Overhead I/O Optimized AI Proxy
🔌
OpenAI Node SDK
Για και τους 3 providers
OpenRouter, Gemini και Ollama υλοποιούν όλοι το OpenAI Chat Completions API. Ένα SDK, τρεις providers — αλλάζει μόνο το baseURL και το apiKey. Αυτό εξαλείφει τρία διαφορετικά SDKs και απλοποιεί δραστικά τη λογική fallback.
Provider Agnostic One SDK baseURL Switch
📂
File System Access API
showOpenFilePicker · showDirectoryPicker
Η FSA API δίνει persistent file handles — αποθηκεύονται στο IndexedDB και ξαναφορτώνουν τα αρχεία αυτόματα στην επόμενη συνεδρία. Επίσης, το showDirectoryPicker επιτρέπει recursive traversal υποφακέλων που είναι αδύνατο με το κλασσικό <input type=file>. Legacy fallback με webkitdirectory για Firefox/Safari.
Persistent Handles IndexedDB Recursive Dirs Legacy Fallback
🗄️
IndexedDB (fileHandleStore)
fileHandleStore.js
Τα File System Access Handles δεν σώζονται στο localStorage — είναι objects, όχι strings. Το IndexedDB είναι ο μόνος browser storage που μπορεί να τα αποθηκεύσει structured. Αυτό επιτρέπει "session restore": κλείνεις browser, ξανανοίγεις, τα αρχεία επανεμφανίζονται αυτόματα.

API Endpoints

4 endpoints — κάθε ένας με διαφορετικό ρόλο στο pipeline. Ο server ποτέ δεν αποθηκεύει Excel αρχεία.

POST /api/analyze
Παίρνει τις πρώτες 30 γραμμές ενός sheet (με style tags: BG, BOLD) και ρωτά το AI να βρει ποια γραμμή είναι column header και πού αρχίζουν τα δεδομένα. Επιστρέφει { headerRows, headerRow, dataStartRow, columns }. Αν το AI αποτύχει (parse error, timeout), κάνει fallback στο heuristic detector (ψάχνει πρώτη γραμμή με 25%+ αριθμούς).

server/index.js — Fallback Engine
async function generateWithOpenRouter(prompt) {
  let lastErr;
  for (const model of FREE_MODELS) {       // 10 models σε σειρά
    try {
      const response = await openrouterClient.chat.completions.create({ model, ... });
      const content = response.choices?.[0]?.message?.content;
      if (!content) { console.warn(`${model} empty, trying next...`); continue; }
      return content;                         // ✓ επιτυχία
    } catch (err) {
      const { isRetryable, isRateLimit } = classifyOpenRouterError(err);
      if (isRateLimit) await sleep(retryAfterMs); // σεβαστός retry-after
      if (!isRetryable) throw err;              // non-retryable: σταματά αμέσως
      lastErr = err;                               // retryable: επόμενο model
    }
  }
  throw finalErr;                              // εξαντλήθηκαν όλα → 429
}
Core Features

Βασικές Λειτουργίες

Οι σημαντικότερες τεχνικές καινοτομίες του project.

🎨
Style-Aware Header Detection
Το AI δέχεται style annotations: [BG:4472C4, BOLD] ανά γραμμή. Η πολύχρωμη επικεφαλίδα είναι το ισχυρότερο signal. Χωρίς αυτό, η ανίχνευση header σε αρχεία με title rows θα ήταν πολύ ανακριβής.
🔄
Heuristic Fallback (χωρίς AI)
Αν το AI αποτύχει, το detectStructure() ψάχνει την πρώτη γραμμή με 25%+ αριθμητικά κελιά. Αυτή είναι σχεδόν πάντα η πρώτη γραμμή δεδομένων. Zero latency, zero API cost, ~80% ακρίβεια.
📚
Few-Shot Learning από Διορθώσεις
Κάθε φορά που ο χρήστης διορθώνει το dataStartRow, αποθηκεύεται η διόρθωση. Οι τελευταίες 5 στέλνονται ως παραδείγματα στο AI prompt. Το AI "μαθαίνει" τις συνήθειες του χρήστη χωρίς fine-tuning.
🔍
Keyword Pre-Filter (χωρίς όριο γραμμών)
Αντί να στέλνει ΟΛΑ τα δεδομένα στο AI, φιλτράρει πρώτα τις γραμμές που περιέχουν keywords. Αφαιρεί stopwords (GR+EN). Το μόνο όριο είναι 40.000 χαρακτήρες — ψάχνει κυριολεκτικά σε εκατοντάδες χιλιάδες γραμμές.
📂
Recursive Folder Loading
Async generator walkDirectory() διατρέχει αναδρομικά υποφακέλους απεριόριστου βάθους. Counter UI "3 / 12 αρχεία" ενημερώνεται live. Drag & drop φακέλου επίσης υποστηρίζεται μέσω FSA API.
📋
Sub-Table Detection
Αν ένα sheet περιέχει πολλούς ξεχωριστούς πίνακες (χωρισμένους με κενές γραμμές), το detectSubTables() τους εντοπίζει. Ο χρήστης επιλέγει ποιον να αναλύσει. Κάθε υποπίνακας αναλύεται ανεξάρτητα.
💾
Session Persistence
Τα file handles αποθηκεύονται στο IndexedDB. Την επόμενη φορά που ανοίγεις την εφαρμογή, εμφανίζεται banner "Βρέθηκαν X αρχεία από προηγούμενη συνεδρία". Ένα κλικ και ξαναφορτώνουν χωρίς να χρειαστεί ξανά file picker.
✏️
Inline Field Name Editing
Κάθε στήλη έχει fieldName (key στο JSON export) και header (original Excel τίτλος). Το fieldName είναι editable inline — hover → εμφανίζεται edit icon → click → input field → Enter ή ✓ για αποθήκευση.
⚡
Rate Limit Handling
Αν όλα τα μοντέλα επιστρέψουν 429, το frontend εμφανίζει countdown 65 δευτερολέπτων ("Επανάληψη σε 45s…") και επαναλαμβάνει αυτόματα. Ο χρήστης βλέπει live countdown χωρίς να κάνει τίποτα.

Βασικοί Αλγόριθμοι

🌲 Recursive Directory Traversal

FileUpload.jsx
async function* walkDirectory(dirHandle) {
  for await (const [name, handle] of
              dirHandle.entries()) {
    if (handle.kind === "file") {
      const ext = name.split(".").pop()
                       .toLowerCase();
      if (EXCEL_EXTS.has(ext)) yield handle;
    } else if (handle.kind === "directory") {
      yield* walkDirectory(handle); // αναδρομή
    }
  }
}

Async generator = lazy evaluation. Δεν φορτώνει όλα τα handles στη μνήμη ταυτόχρονα. Απεριόριστο βάθος υποφακέλων.

🧮 Heuristic Structure Detection

server/index.js
function findDataStartByNumeric(cells, max) {
  for (let i = 0; i < max; i++) {
    const nonEmpty = cells[i].filter(
      c => cellVal(c) !== "");
    if (!nonEmpty.length) continue;
    const numeric = nonEmpty.filter(
      c => typeof c.value === "number");
    // 25%+ αριθμοί = γραμμή δεδομένων
    if (numeric.length >= Math.ceil(
          nonEmpty.length * 0.25))
      return i;
  }
  return null;
}

Το 25% threshold είναι εμπειρικό — αρκετά χαμηλό για sparse data, αρκετά ψηλό για να αγνοεί metadata rows.

🔍 Keyword Pre-Filter για Search

server/index.js — filterRelevantRows()
function filterRelevantRows(cells, question) {
  const stopWords = new Set(["το", "τα", "και", "the", "and", /* ... */]);
  const keywords = question.toLowerCase()
    .split(/[\s,;:.!?()\[\]]+/)
    .filter(w => w.length > 2 && !stopWords.has(w));

  if (keywords.length === 0)
    return cells.map((row, idx) => ({ idx, row })); // όλες οι γραμμές

  return cells.flatMap((row, i) => {
    const text = row.map(c => String(c.formattedValue || ""))
                    .join(" ").toLowerCase();
    return keywords.some(kw => text.includes(kw))
      ? [{ idx: i, row }] : [];      // χωρίς όριο γραμμών
  });
}

Design Decisions

Βασικές Αρχιτεκτονικές Αποφάσεις

Οι επιλογές που δεν είναι προφανείς χωρίς context.

Απόφαση #1
Client-side Excel parsing — όχι server
Το Excel parsing γίνεται 100% στον browser με SheetJS. Ο server ποτέ δεν λαμβάνει το αρχείο — μόνο text cells. Αυτό σημαίνει: (α) μηδενικό upload time για μεγάλα αρχεία, (β) privacy by design — τα αρχεία δεν φεύγουν από τον υπολογιστή, (γ) δεν χρειάζεται disk storage στον server.
Απόφαση #2
Ένα OpenAI SDK για τρεις providers
Αντί για ξεχωριστά SDKs (google-generativeai, ollama-js), όλοι οι providers εκθέτουν OpenAI-compatible endpoint. Ένα SDK, τρία clients με διαφορετικό baseURL. Αυτό σημαίνει zero-cost provider switch και ένα μέρος για fallback logic.
Απόφαση #3
@e965/xlsx fork αντί για επίσημο SheetJS
Η επίσημη SheetJS κρύβει τα cell styles (bgColor, bold) στις νέες εκδόσεις πίσω από subscription. Το community fork @e965/xlsx τα δίνει δωρεάν. Αυτά τα styles είναι κρίσιμα για ακριβή header detection — χωρίς αυτά θα έπρεπε να βασίζεσαι μόνο στη θέση των κελιών.
Απόφαση #4
Keyword filter αντί για σταθερό row limit
Αρχικά υπήρχε σταθερό όριο 60 γραμμών ανά sheet. Αντικαταστάθηκε με keyword pre-filter που αφαιρεί άσχετες γραμμές. Έτσι 10.000 γραμμές → ίσως 50 relevant → στο AI. Το αποτέλεσμα είναι ταυτόχρονα πιο ακριβές (λιγότερος θόρυβος) και πιο πλήρες (καμία γραμμή χάνεται).
Απόφαση #5
IndexedDB για file handles — όχι localStorage
Τα FSA File Handles είναι JavaScript objects. Το localStorage αποθηκεύει μόνο strings — δεν μπορεί να serialized ένα FileSystemFileHandle. Το IndexedDB υποστηρίζει structured cloning και είναι ο μόνος τρόπος για persistent handles και session restore.
